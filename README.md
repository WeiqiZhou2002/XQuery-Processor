# CSE 232B - XQuery Processor

Course project for building an XQuery processor from scratch.
This repo currently targets **Milestone 1: an XPath evaluator**.

## Project Layout

```
.
├── main/                 # Java package used by the build and grader
│   ├── Main.java
│   ├── XPathEngine.java
│   └── antlr/
│       ├── XPath.g4
│       └── XPath*.java   # regenerated from XPath.g4
├── data/                 # sample XML inputs
├── docs/                 # course notes and reference PDFs
├── outputs/              # local query/test outputs
├── tools/                # local build tools, including the ANTLR jar
├── dist/                 # generated submission archives
├── legacy/root-copies/   # preserved old root-level source copies
├── Makefile
└── README.md
```

The authoritative source files are under `main/`. The files in
`legacy/root-copies/` are old root-level copies kept for reference only.

## Architecture

```
XQuery string -> ANTLR parser -> AST
Input XML doc -> DOM parser   -> XPathEngine -> DOM nodes -> serializer -> output XML
```

| Piece | File(s) | Notes |
| --- | --- | --- |
| XPath grammar | `main/antlr/XPath.g4` | Edit this when changing grammar rules. |
| XQuery grammar | `main/antlr/XQuery.g4` | Milestone 2 scaffold; includes embedded XPath rules. |
| Generated parsers | `main/antlr/XPath*.java`, `main/antlr/XQuery*.java` | Regenerated by `make parser`. Do not hand-edit. |
| Engine | `main/XPathEngine.java` | Core XPath evaluation logic. |
| XQuery engine | `main/XQueryEngine.java` | Milestone 2 learning scaffold with TODOs by semantic rule. |
| Driver/serializer | `main/Main.java` | Wires parser, engine, and output serialization. |
| Sample XML | `data/*.xml` | Inputs for local runs and course examples. |

## Build And Run

```bash
make
make run Q='doc("data/test.xml")/library/book'
make test
make submission
```

Useful targets:

| Target | What it does |
| --- | --- |
| `make` | Regenerates the ANTLR parser if needed and compiles into `build/classes`. |
| `make parser` | Regenerates both XPath and XQuery ANTLR parser files. |
| `make xpath-parser` | Regenerates `main/antlr/XPath*.java` from `main/antlr/XPath.g4`. |
| `make xquery-parser` | Regenerates `main/antlr/XQuery*.java` from `main/antlr/XQuery.g4`. |
| `make run` | Runs one query against `XML`, writing to `OUT`. |
| `make test` | Runs the five course example queries against `data/j_caesar.xml`. |
| `make submission` | Builds `dist/submission.zip` in the grader layout. |
| `make clean` | Removes generated build/output/submission artifacts. |

Examples:

```bash
make run XML=data/test.xml Q='doc("data/test.xml")/library/book' OUT=outputs/books.xml
make run XML=data/j_caesar.xml Q='doc("data/j_caesar.xml")//PERSONA' OUT=outputs/personae.xml
```

## Notes

- `tools/antlr-4.13.2-complete.jar` is checked in so the project builds without downloading ANTLR.
- `outputs/`, `build/`, and `dist/submission.zip` are generated artifacts.
- `legacy/root-copies/` exists only to avoid losing earlier duplicate files while keeping the root tidy.
- The current command-line driver still runs the Milestone 1 XPath engine. For Milestone 2, start by filling in `XQueryEngine.java`, then switch `Main.java` to use `XQueryLexer`, `XQueryParser`, `parser.xq()`, and `XQueryEngine`.
